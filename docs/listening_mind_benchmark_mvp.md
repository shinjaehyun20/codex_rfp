# Listening Mind 벤치마킹 기반 고객 검색 데이터 분석 플랫폼 개발안

## 1) 목표 및 기대 효과
- **고객 응대 품질 모니터링/개선**: 검색 의도와 VOC를 연결해 상담 스크립트, FAQ, 챗봇 답변을 개선.
- **고객 인사이트 확보**: 주제·의도·페르소나·여정을 통합 분석하여 제품/마케팅/CS 의사결정 지원.
- **이슈 기반 대응 전략**: 급증 키워드, 부정 감성, 이탈 경로를 조기 탐지해 대응 우선순위 도출.

---

## 2) 리스닝마인드 허블 벤치마크 포인트 → 구현 매핑

### A. 인텐트(의도) 파악
- 벤치마크 핵심: 대규모 검색 데이터에서 관심사/고민/욕구 구조화.
- 구현:
  - 수집: Google/Naver 검색어 + 내부 문의 키워드.
  - 분석: 형태소/문장 임베딩 + 의도 분류(탐색/비교/구매/불만 등).
  - 산출: 의도별 검색량, 증감률, 감성 비중.

### B. 고객 여정 시각화(최대 10단계)
- 벤치마크 핵심: 검색 경로를 단계별로 추적해 전환/이탈 파악.
- 구현:
  - 세션화: 시간창 + 사용자 식별자(익명).
  - 경로 그래프: 단계 전이 확률, 대표 경로 Top-N.
  - 제약: 단계 수 10단계 상한.

### C. 키워드 클러스터링/분석
- 벤치마크 핵심: 유사 의도 키워드를 군집화해 SEO/콘텐츠 전략 지원.
- 구현:
  - 임베딩 기반 군집(HDBSCAN/KMeans 하이브리드).
  - 클러스터 라벨링(LM 요약 + 규칙 기반 보정).
  - 연관어 및 롱테일 추출.

### D. 페르소나 분석
- 벤치마크 핵심: 검색행동 기반 세그먼트로 타겟 전략 정교화.
- 구현:
  - 특성: 의도 분포, 여정 길이, 감성, 이슈 관심사.
  - 산출: 페르소나 카드(핵심 니즈, 장벽, 추천 메시지).

### E. 생성형 AI 분석
- 벤치마크 핵심: 자동 요약/인사이트 도출로 분석 시간 단축.
- 구현:
  - LLM 요약 체인: 주간 변화, 이슈 원인 가설, 실행 권고안 생성.
  - 가드레일: 출처 근거(키워드/지표) 포함, 환각 방지 템플릿.

## 3) MVP 기능 정의 (12주)

### 3.1 데이터 파이프라인
1. 수집 Ingestion
   - Google/Naver 검색 데이터(일 단위), 내부 VOC(상담/챗봇).
2. 정제/표준화
   - 금칙어/PII 마스킹, 언어 정규화, 중복 제거.
3. 분석
   - 의도 분류, 키워드 군집, 여정 그래프, 페르소나 세그먼트.
4. 인사이트
   - LLM 기반 요약 리포트 + 우선순위 액션 제안.

### 3.2 대시보드
- 실시간(near-real-time) 트렌드 패널.
- 의도 분포/증감, 이슈 급증 알림.
- 10단계 여정 Sankey/Path 뷰.
- 페르소나 카드 및 CS 개선 권고.

### 3.3 운영 기능
- 알림: 슬랙/이메일(급등/급락/부정감성 임계치).
- 리포트: 주간 PDF/슬라이드 익스포트.
- 권한: 관리자/분석가/열람자 RBAC.

---

## 4) 기술 아키텍처 제안
- **Ingestion**: API Connector + Batch Worker(Celery)
- **Storage**: Raw(Object Storage) + Feature(OLAP/warehouse)
- **Serving API**: FastAPI
- **AI/ML**: 임베딩 모델 + 의도 분류기 + LLM 요약
- **Frontend**: Next.js 대시보드
- **Observability**: 로그/메트릭/데이터 품질 모니터링

현재 monorepo(`apps/api`, `apps/worker`, `apps/web`) 구조를 그대로 활용해 단계적으로 구현 가능.

---

## 5) 데이터 계약(Contract) 확장
MVP 산출물로 `customer.intent.snapshot.v1` 스키마를 추가:
- 기간(`period`), 채널(`channels`)
- 의도(`intents`), 여정(`journey_paths`, 최대 10단계)
- 페르소나(`personas`)
- 생성형 AI 인사이트(`generated_insights`)

이 스냅샷은 API 응답/리포트/알림의 공통 원본으로 사용.

---

## 6) KPI 및 검증 지표
- 분석 품질: 의도 분류 F1, 군집 일관성(silhouette), 요약 유효성(휴먼 평가).
- 비즈니스: CS 해결시간 단축, 재문의율 감소, FAQ 클릭률/해결률 상승.
- 운영: 데이터 지연시간, 실패율, 알림 정탐률.

---

## 7) 단계별 로드맵
1. **Phase 1 (0~4주)**: 수집/정제 + 기본 의도 대시보드.
2. **Phase 2 (5~8주)**: 여정 시각화 + 키워드 클러스터링 + 알림.
3. **Phase 3 (9~12주)**: 페르소나 + 생성형 AI 인사이트 + 운영 안정화.
